# Task pd-003: Show presigned URLs for file downloads

## Objective
Add presigned URL generation for files so users can download them directly from the UI.

## Implementation Plan

### 1. Backend - Storage Client Enhancement
- **File**: `backend/internal/storage/client.go`
- Add `GetPresignedURL` method that:
  - Takes bucket, key, and expiration duration
  - Returns a presigned GET URL from S3
  - Uses AWS SDK v2 presigner
  
### 2. Backend - DTO Update
- **File**: `backend/internal/api/dto/files.go`
- Update `FileSystemEntry` struct to include optional `presigned_url` field
- Add new struct: `PresignedURLResponse` for direct URL endpoint

### 3. Backend - HTTP Handler
- **File**: `backend/internal/files/http.go`
- Add new handler: `GetPresignedURL(c *gin.Context)` for GET endpoint
  - Path: `GET /api/v1/files/presigned-url`
  - Query params: `path` (file path)
  - Query params: `expiration` (optional, in minutes, default 60)
  - Returns presigned URL
- Alternatively, include presigned URLs in the List endpoint

### 4. Frontend - OpenAPI Codegen
- Run `npm run codegen` to regenerate client types

### 5. Frontend - Dashboard Page
- **File**: `frontend/src/pages/dashboard-page.tsx`
- Add download button/link for files using presigned URL
- Add icon (Download from lucide-react) next to each file
- Show presigned URL in a tooltip or modal on click

## Implementation Details

### Approach Decision: Include in List Response vs Separate Endpoint
**Selected**: Include presigned URLs in the List response for efficiency
- Reduces API calls
- Keeps file listing atomic
- Presigned URLs are cheap to generate

### Security Considerations
- Presigned URLs expire after 1 hour (configurable)
- Only allow reading (GET) operations
- Validate that the requesting user owns the file's bucket

- [ ] Clicking download button opens/downloads file
- [ ] Presigned URLs work with browser and external tools (curl, wget)
- [ ] URLs respect the expiration time

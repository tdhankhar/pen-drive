# Code Plan: pd-002 - Pre-commit Hook for Frontend-Only Formatting

**Issue**: Configure pre-commit to trigger frontend formatting only when frontend files are staged, avoiding unnecessary backend reformatting on frontend-only commits.

**Status**: open | **Priority**: p1 | **Kind**: task

---

## Current State Analysis

### Root `.pre-commit-config.yaml`
- Only has `gofmt` for backend Go files (`^backend/.*\.go$`)
- Uses `files` pattern to scope to backend
- No frontend hooks configured at root level

### Frontend `.pre-commit-config.yaml`
- Defines `frontend-lint` hook that runs `npm run lint`
- Uses `always_run: true` and `pass_filenames: false` — **PROBLEM**: This runs on every commit regardless of staged files
- No formatting (only linting)
- Missing `oxfmt` for code formatting

### Frontend `package.json`
- Has `fmt: "oxfmt"` script for formatting
- Has linting with `oxlint` and `eslint`
- No pre-commit filtering by file type

---

## Problem Statement

**Current Behavior**:
- Frontend lint hook runs on ALL commits, even backend-only commits
- No formatting step in pre-commit for frontend
- Backend Go files are formatted but frontend files are not

**Desired Behavior**:
- Frontend hooks trigger ONLY when `frontend/**` files are staged
- Frontend formatting with `oxfmt` before linting
- Both hooks run efficiently without cross-domain interference

---

## Implementation Plan

### Phase 1: Update Root `.pre-commit-config.yaml`

**Goal**: Add frontend hooks with proper file scoping

**Tasks**:

1. Add `frontend-fmt` hook
   - Entry: `npm run fmt` (oxfmt)
   - Language: `system`
   - Files: `^frontend/.*\.(ts|tsx|jsx|js|json)$`
   - Stages: `pre-commit`

2. Add `frontend-lint` hook  
   - Entry: `npm run lint` (oxlint + eslint)
   - Language: `system`
   - Files: `^frontend/.*\.(ts|tsx|jsx|js|json)$`
   - Stages: `pre-commit`

3. Ensure `gofmt` hook remains for backend

**Why**:
- Consolidate all hooks in root config
- Use `files` pattern to trigger only on frontend changes
- Separate formatting (pre-lint) from linting (error checking)
- No `always_run: true` — hooks only trigger if files match pattern

### Phase 2: Remove Redundant Frontend Config

**Goal**: Clean up duplication

**Tasks**:
1. Remove or archive `frontend/.pre-commit-config.yaml`
2. All hooks managed from root `.pre-commit-config.yaml`

**Why**:
- Single source of truth for hooks
- Reduces confusion and maintenance burden
- Standard practice for monorepos

### Phase 3: Testing & Validation

**Goal**: Verify hooks work as intended

**Test Cases**:
1. Stage only backend files → gofmt runs, frontend hooks skip
2. Stage only frontend files → frontend-fmt and frontend-lint run, gofmt skips
3. Stage mixed files → all applicable hooks run
4. Run `pre-commit run --all-files` → all hooks execute

**Commands**:
```bash
# Test specific hook
pre-commit run frontend-fmt --all-files
pre-commit run gofmt --all-files

# Test selective run
git add frontend/src/App.tsx
pre-commit run  # should run frontend hooks only

git add backend/cmd/api/main.go  
pre-commit run  # should run gofmt only
```

## File Changes

### `.pre-commit-config.yaml` (Root)

**From**:
```yaml
default_install_hook_types:
  - pre-commit

repos:
  - repo: local
    hooks:
      - id: gofmt
        name: gofmt
        entry: gofmt -w
        language: system
        files: ^backend/.*\.go$
        stages:
          - pre-commit
```

**To**:
```yaml
default_install_hook_types:
  - pre-commit

repos:
  - repo: local
    hooks:
      # Backend formatting
      - id: gofmt
        name: gofmt
        entry: gofmt -w
        language: system
        files: ^backend/.*\.go$
        stages:
          - pre-commit

      # Frontend formatting (oxfmt)
      - id: frontend-fmt
        name: frontend format (oxfmt)
        entry: bash -c "cd frontend && npm run fmt"
        language: system
        files: ^frontend/.*\.(ts|tsx|jsx|js|json)$
        stages:
          - pre-commit

      # Frontend linting (oxlint + eslint)
      - id: frontend-lint
        name: frontend lint (oxlint + eslint)
        entry: bash -c "cd frontend && npm run lint"
        language: system
        files: ^frontend/.*\.(ts|tsx|jsx|js|json)$
        stages:
          - pre-commit
```

### `frontend/.pre-commit-config.yaml`

**Action**: Delete (consolidate to root)

## Considerations & Edge Cases

1. **CD into frontend directory**: Hooks must run `npm` commands from `frontend/` directory
   - Solution: Use `bash -c "cd frontend && npm run X"`

2. **File pattern for frontend**: Match TS/TSX/JS/JSON but exclude node_modules, dist
   - Pattern: `^frontend/.*\.(ts|tsx|jsx|js|json)$` (negative patterns handled by git)

3. **Formatting order**: oxfmt must run before oxlint to catch formatted code
   - Solution: Two separate hooks, gofmt runs separately

4. **Performance**: npm install happens in frontend
   - Should be fast since dependencies cached

5. **Hook failures**: If formatting changes files, pre-commit auto-stages them
   - Behavior: Expected, files reformatted automatically
